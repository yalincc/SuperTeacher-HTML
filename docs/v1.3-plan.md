## 一、可用技术栈总览

### 1.1 基础能力

| 技术 | 浏览器支持 | 零依赖 | 适用场景 |
|------|:---------:|:-----:|---------|
| CSS `@keyframes` + `animation` | 全平台 | ✅ | 循环动画、入场动画 |

| CSS `@starting-style` | Chrome 117+ | ✅ | mount 入场动画（无需 JS） |
| `interpolate-size: allow-keywords` | Chrome 129+ | ✅ | `height: auto` 动画 |
| `IntersectionObserver` | 全平台 | ✅ | 滚动触发出场 |
| View Transitions API | Chrome 111+ | ✅ | 页面切换过渡 |
| `animation-timeline: view()` | Chrome 115+ | ✅ | 滚动驱动动画（纯 CSS） |
| CSS `backdrop-filter` | 全平台 | ✅ | 毛玻璃效果 |
| CSS `conic-gradient` | 全平台 | ✅ | 环形进度图 |
| SVG inline | 全平台 | ✅ | 树形连线、简单图形 |
| `prefers-reduced-motion` | 全平台 | ✅ | 无障碍：尊重用户动效偏好 |
| `prefers-color-scheme` | 全平台 | ✅ | 暗色模式自动检测 |

### 1.2 不可用 / 不建议的

| 技术 | 原因 |
|------|------|
| Framer Motion | 第三方依赖，违反零依赖原则 |
| GSAP / Anime.js | 同上 |
| Lottie / Rive | 需要额外资源文件 |
| CSS `round()` / `mod()` | Chrome 125+，兼容性不足 |
| Web Animations API (WAAPI) | 可用但非必需，CSS 能覆盖 90% 需求 |

## 二、动画技术方案详解

### 2.1 CSS @keyframes — 自定义动画库

在 `index.css` 中用 `@theme` 注入 Tailwind 可直接使用的动画：

```css
@import "tailwindcss";

@theme {
  --animate-fade-in: fade-in 0.4s ease-out;
  --animate-slide-up: slide-up 0.4s ease-out;
  --animate-slide-down: slide-down 0.3s ease-out;
  --animate-pulse-once: pulse-once 0.5s ease-out;
  --animate-shake: shake 0.4s ease-in-out;
  --animate-bounce-in: bounce-in 0.5s cubic-bezier(0.68, -0.55, 0.265, 1.55);
}

@keyframes fade-in {
  from { opacity: 0; }
  to   { opacity: 1; }
}

@keyframes slide-up {
  from { opacity: 0; transform: translateY(12px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes slide-down {
  from { opacity: 0; transform: translateY(-8px); max-height: 0; }
  to   { opacity: 1; transform: translateY(0); max-height: 500px; }
}

@keyframes pulse-once {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.04); }
  100% { transform: scale(1); }
}

@keyframes shake {
  0%, 100% { transform: translateX(0); }
  20%      { transform: translateX(-4px); }
  40%      { transform: translateX(4px); }
  60%      { transform: translateX(-3px); }
  80%      { transform: translateX(3px); }
}

@keyframes bounce-in {
  0%   { opacity: 0; transform: scale(0.9); }
  60%  { transform: scale(1.02); }
  100% { opacity: 1; transform: scale(1); }
}
```

然后组件里直接用 Tailwind class：`animate-fade-in`、`animate-slide-up` 等。

**优点**：一次定义，全局复用，语义化 class 名
**局限**：动画只触发一次（mount 时），无法滚动驱动

---

### 2.2 @starting-style — 纯 CSS mount 动画

Tailwind v4 原生支持。用于组件首次挂载时的入场效果：

```css
/* 知识点面板 */
.section-panel {
  @starting-style {
    opacity: 0;
    max-height: 0;
  }
  opacity: 1;
  max-height: 2000px;
  transition: opacity 0.3s, max-height 0.5s ease-out;
}
```

配合 `interpolate-size: allow-keywords`，可以从 `height: 0` 过渡到 `height: auto`：

```css
.knowledge-collapsible {
  interpolate-size: allow-keywords;
  overflow: hidden;
  transition: height 0.4s ease-out;
}
```

**优点**：零 JS，mount/unmount 自动动画
**局限**：仅 Chrome 117+，需备选 fallback

### 2.3 IntersectionObserver — 滚动驱动的逐段淡入

```tsx
// hooks/useScrollReveal.ts
function useScrollReveal() {
  const ref = useRef<HTMLDivElement>(null)
  const [visible, setVisible] = useState(false)

  useEffect(() => {
    const el = ref.current
    if (!el) return
    const obs = new IntersectionObserver(
      ([entry]) => { if (entry.isIntersecting) setVisible(true) },
      { threshold: 0.1 }
    )
    obs.observe(el)
    return () => obs.disconnect()
  }, [])

  return { ref, visible }
}

// 组件里
// <div ref={ref} className={visible ? 'animate-slide-up' : 'opacity-0'}>{content}</div>
```

**优点**：逐段浮现，长页面阅读体验大幅提升
**局限**：需要 JS，每个块一个 observer 实例

---

### 2.4 View Transitions API — 页面切换动画

React Router 页面跳转时加过渡：

```tsx
// 点击课时卡片 → 过渡到课时页
function navigateWithTransition(url: string) {
  if (document.startViewTransition) {
    document.startViewTransition(() => {
      router.navigate(url)
    })
  } else {
    router.navigate(url)  // fallback
  }
}
```

```css
/* index.css */
::view-transition-old(root) {
  animation: fade-out 0.2s ease-out;
}
::view-transition-new(root) {
  animation: slide-up 0.3s ease-out;
}
```

**优点**：API 原生，页面间切换丝滑
**局限**：仅 Chrome 111+，非 Chrome 静默降级

---

### 2.5 CSS Confetti — 无依赖庆祝粒子

纯 CSS `@keyframes` + 多个 `::before` / `::after` 伪元素：

核心思路：
- 用 `box-shadow` 生成多个彩色圆点
- 用 `@keyframes` 控制散开/下落
- 用 `animation-delay` 制造逐粒效果
- 总共 ~50 行 CSS，零 JS

**优点**：零依赖，体积极小
**局限**：粒子数量受限于 box-shadow 数量（一般 20~30 个够用）

## 三、视觉润色技术方案

### 3.1 毛玻璃导航栏 (Glassmorphism)

```css
.nav-glass {
  background: rgba(255, 255, 255, 0.8);
  backdrop-filter: blur(12px);
  -webkit-backdrop-filter: blur(12px);
  border-bottom: 1px solid rgba(0, 0, 0, 0.05);
}
```

**优点**：现代感强，纯 CSS
**适用**：TopNav、BottomNav

---

### 3.2 环形进度图 (Conic Gradient)

纯 CSS 实现，不依赖 SVG 或 Canvas：

```css
.progress-ring {
  background: conic-gradient(
    #3b82f6 calc(var(--progress) * 1%),
    #e5e7eb 0
  );
  border-radius: 50%;
  mask: radial-gradient(transparent 58%, black 60%);
}
```

**优点**：零 JS、零额外 DOM、HTML 一个 div 搞定
**适用**：首页统计区

---

### 3.3 SVG 树形连线 — 替代 ●/○

用 inline SVG 画垂直连接线 + 节点圆：

```svg
<svg width="20" height="100%">
  <line x1="10" y1="0" x2="10" y2="100%" stroke="#d1d5db" stroke-width="2"/>
  <circle cx="10" cy="50%" r="6" fill="#3b82f6"/>
</svg>
```

配合 CSS `position: absolute` 放在文字左侧。

---

### 3.4 提示框渐变条

```css
.callout-warning {
  border-left: 4px solid #ef4444;
  background: linear-gradient(90deg, #fef2f2 0%, #fff 40%);
}
```

**优点**：一行 CSS，视觉层次立现

---

### 3.5 暗色模式

两种策略可选：

| 策略 | 用法 | 优点 | 缺点 |
|------|------|------|------|
| `prefers-color-scheme` | `@media (prefers-color-scheme: dark)` | 自动跟随系统 | 用户无法手动切换 |
| Tailwind `dark:` | `className="dark:bg-gray-900"` | 可手动切换 | 需 JS toggle + localStorage |

**推荐**：先做 `prefers-color-scheme` 自动模式，成本最低。后续可加手动 toggle。

## 五、风险与取舍

| 风险 | 应对 |
|------|------|
| `@starting-style` / `interpolate-size` 浏览器兼容 | 仅 Chrome 129+，其他浏览器静默降级：折叠/展开功能正常，只是没动画 |
| View Transitions API 兼容 | 非 Chrome 自动 fallback 到无过渡跳转 |
| IntersectionObserver 性能 | 10 个 observer 以内无压力，每课大约 5~15 个 blocks |
| 动画过多导致眩晕 | 所有动画遵守 `prefers-reduced-motion`，用户开启"减弱动态效果"后全部禁用 |

## 六、不做的

| 事项 | 原因 |
|------|------|
| 引入 Framer Motion / GSAP | 违反零依赖原则 |
| Canvas 粒子系统 | 过度工程，CSS confetti 够用 |
| 3D transform 动画 | 教育产品不需要，保持克制 |
| 音频反馈 | 增加复杂度，不在 v1.3 范围 |
| PWA / Service Worker | v1.2 roadmap 已列入 P2，不抢跑 |
