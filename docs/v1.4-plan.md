# P0.2 前端内联渲染修复计划

> 状态：✅ 已完成并验收通过 | 创建时间：2026-06-15 | 完成时间：2026-06-15

## 问题概述

浏览器逐课验收发现：段落文本中的 `$...$` 行内 LaTeX（化学式如 `$O_2$`、方程式如 `$\xrightarrow{点燃}$`）和 `**加粗**` 未被渲染，直接显示原始标记。

**根因**：`renderInline.tsx` 只处理 bold/italic，未集成 KaTeX 行内渲染；多个组件未调用 renderInline。

---

## 修复范围（8 项子任务）

### 任务 1：renderInline.tsx 增加 $...$ 行内 LaTeX 渲染 ⭐ 核心

**文件**: `src/utils/renderInline.tsx`

**现状**：只处理 `**bold**` 和 `*italic*`

**改动**：
- KaTeX 已安装（`package.json` 中 `"katex": "^0.16.22"`，`@types/katex` 也已安装）
- 在现有正则基础上，增加 `$...$` 行内 LaTeX 匹配
- 用 `katex.renderToString()` 将 LaTeX 渲染为 HTML
- 正则顺序：先匹配 `$$...$$`（display 模式），再匹配 `$...$`（inline 模式），再 `**bold**`，再 `*italic*`
- KaTeX 的 `throwOnError: false` 容错，渲染失败时显示原始文本

**技术说明**：
- KaTeX 是一个轻量级数学公式渲染库，`renderToString()` 可在服务端/客户端将 LaTeX 字符串转为 HTML
- `$...$` 是行内公式标记，`$$...$$` 是独立块级公式标记
- 化学式示例：`$O_2$` → 渲染为带下标的 O₂
